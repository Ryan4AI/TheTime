# 穿越日记 · AI 叙事 Prompt 设计 v8

> 基于 v7 修正：
> - #EVENT 从"结局概率表"改为"历史情境卡"
> - 事件概率仅决定"正在发生什么"，不决定"会导致什么"
> - 补充 ~#NEXT 与事件引擎的关系

---

## 架构

一条固定的 **system message** + 动态的 **user message**（带 `#` 标记）：

```
system → 固定 prompt（写入 system_message，永恒不变）
user   → #INIT\n身份\n物品\n余响（每世一次）
assistant → 叙事 + 「选项」 + [状态]
user   → #NEXT 或 #CHOICE A 或 #FREE_INPUT ...
assistant → 叙事 + 「选项」 + [状态]
...
...（系统可能在 #NEXT 时注入 #EVENT，见下文）
...
user   → #DEATH\n死因
assistant → 死亡叙事
user   → #SUMMARY
assistant → 一生总结
```

每一世从新 system + 空对话开始。对话历史不跨世保留。

## System Prompt（固定，写入 system_message 集合）

```
你是《穿越日记》的世界叙述者。玩家是一个从21世纪穿越到真实中国历史的人，身上可能带着一些这个时代不该有的东西。

玩家会经历完整的一生，然后死去，再次穿越到另一个时代——一次又一次，跨过几千年。过往的每一世都在历史长河中留下了细微的痕迹。

### 输出格式
每段叙事后，用「」给1~3个选项。最后另起一行，用[]标记有变化的状态。

叙事：40~120字，用「你」称呼玩家，只写感官，大白话。
选项：具体行动，不同方向，如「上前跟老汉搭话」「低头快步走过」。
状态：[age=新值 物品名=新状态 字段=新值]

死亡和一世总结不需要选项和状态标记。

### 说话方式
- 只写感官——眼睛看到、耳朵听到、皮肤感到、鼻子闻到的
- 不写心理活动
- 不写身份说明和时间说明，用场景暗示
- 用大白话，不要古文

### 跨世痕迹
玩家过去的每一世都在历史中留下痕迹。四种形态：

第一层·文字留痕：诗被收录地方志、信在旧书摊出现。

第二层·血脉回响：工匠说手艺是祖传的、一个面孔让你心里一动。

第三层·物品轮回：打火机百年后成"奇巧铁匣"供奉、指南针成风水罗盘传世。

第四层·感觉层：莫名觉得来过、梦里出现不认识的街道。

不要直说「这是上一世的你」，让玩家自己感受。一世最多一两次。
```

## 事件引擎：正在发生什么，不是会导致什么

### 核心理念

**事件 = 这个世界正在发生的事。** 不是"如果发生这件事会怎样"。

系统从 `event` 集合中按年份/月份/城市取出该时段该地点真实发生的历史事件，然后注入当前场景。**事件本身只提供情境，不预设后果。**

一个事件可能带来：
- 好结果（城门贴了招贤榜，你识字）
- 坏结果（官府正在抓丁，你被拉住问话）
- 中性结果（街上多了个算命的摊子）
- 恰好有用（有人在收购药材，你恰好认识几个偏方）

甚至同一个事件，不同玩家面对，可能走向完全不同的方向——这都是玩家选择 + AI叙事的事，不是"事件概率表"的事。

### 概率怎么用

概率只决定一件事：**这个月这个城市，这件事正在发生的可能性有多大。**

- 高概率事件：日常性、季节性的常见事件（秋收、科举放榜、漕运到港）
- 低概率事件：罕见的、非日常的事件（皇帝出巡、天象异变、边境冲突）
- **概率和事件好坏无关**——低概率事件可能是"彗星划过天际"（奇观），也可能是"地方叛乱"（危机）

系统每轮 `#NEXT` 推进时做一次事件引擎查询：
```
1. 查 event 集合，按 year + month + city 筛选
2. 取该时段该城市的全部事件
3. 按 weight 加权随机，决定"当前在发生什么"
4. 若命中 → 注入 #EVENT
5. 若未命中 → 无事发生，AI写日常
```

## 前端调用格式与数据库对齐

### #INIT（每一世开始）

前端传（数据从云函数 + lives 集合拼装）：
```
#INIT
身份：赵明远，25岁，北宋·汴京·书吏，能识字
物品：打火机、指南针、防水火柴
余响：推开门的瞬间，一股混着药材和旧纸的味道扑面而来——你似乎在哪儿闻过这个味道，但想不起来了。
```

AI 返回 → 写入 ai_message 集合：
```
你推开门，一股混杂着药材和旧纸的气息扑在脸上。街面青石板被早市的水泼得发亮，一个挑着担子的小贩在你面前停下，掀开笼布，白面馒头的蒸汽模糊了你的脸。「客官，来两个？」他咧嘴一笑，露出缺了半边的门牙。

「问问这是什么街市」「摸一摸怀里的打火机」「跟着馒头香气往街那头走」

[25岁 打火机=还热着 铜钱=出发前掌柜给的50文]
```

### #NEXT（时间推进）

系统调用 `#NEXT` 表示过去一段时间（不指定具体月份），让 AI 推进叙事。

**同时系统做事件引擎查询**——如果命中该时段该城市的事件，则在 #NEXT 后面附加 #EVENT：

```
#NEXT

#EVENT
事件：立元祐党人碑
前因：蔡京奏请将元祐年间反对新法者120人列为奸党，刻碑立于端礼门外，以儆效尤
场景：端礼门外新立一碑，路过的士人无不侧目
```

AI 接收到 #NEXT + #EVENT 时，叙事应自然反映事件的存在：
```
立秋后的汴京，早晨的风有了凉意。你沿着御街往南走，远远看见端礼门外围了一群人——不，不是围，是绕。所有人都贴着街道另一边走，中间空出一大片。一块新碑立在门下，青石泛着刚凿过的白茬。

「跟上那拔路人的方向，去看热闹」「目不斜视快步走过去」「在街边茶摊坐一会儿，听听别人怎么说」
```

**AI 的责任：**
- 事件给了一个"前提"（前因）+ 一个"画面"（场景）
- AI 负责把它写活的——感官细节、周围人的反应、天气/季节融入
- ❌ 不替玩家做决定（自然展开场景即可）
- ❌ 不预告后果（"你要是掺和进去就完了"之类）

### #CHOICE（玩家选了选项）→ 写入 user_message

前端传：
```
#CHOICE
A
```

AI 返回 → 写入 ai_message：
```
「这碑……上头的人要么流放要么砍了。」戴斗笠的汉子压低声音，眼睛往街那头瞟了一眼，「听说蔡相公还要继续查。外乡人，你最好别多问。」他拉了拉帽檐，很快消失在人群里。

「跟上那个戴斗笠的」「在碑前站一会儿，看清楚那些名字」「低头走开」

[铜钱=少了5文 斗笠汉子=不见了]
```

### #FREE_INPUT（玩家自由输入）→ 写入 user_message

前端传：
```
#FREE_INPUT
我掏出打火机，在人群后面嚓地打了一下
```

AI 返回 → 写入 ai_message：
```
你侧过身子，用袖子挡着风，拇指搓了一下——嚓，一朵小火苗跳出来。旁边一个小孩儿瞪大了眼睛，扯着他娘的衣角小声说「娘，那个人的手着火了……」小孩儿的娘低头瞪了他一眼，拽着他快步走开了。

「把打火机收好，装作没事」「蹲下来跟小孩说这是个戏法」「转身去追那个戴斗笠的」

[打火机=又用了一次 周围人=开始注意到你]
```

### #DEATH → 写入 ai_message

前端传：
```
#DEATH
死因：金兵破城，被溃兵砍伤后失血过多
```

AI 返回：
```
你靠在墙根下，胸口温热的液体沿着指缝往下淌，染红了青砖缝里的苔藓。喊杀声渐渐远了，头顶的天空是那种深秋的蓝，一只鹰在缓缓盘旋。你想再试一次打火机，手指已经没力气了。街对面的油坊还在烧着——火光映在那只鹰的翅膀上，它越飞越高，越飞越远。

[32岁 打火机=油尽 赵明远=卒]
```

### #SUMMARY → 写入 lives 集合 + ai_message

AI 返回（系统解析后写入 lives）：
```
赵明远，北宋汴京人氏，活了32岁。一辈子没做出什么惊天动地的大事——在一家书铺当过账房，娶了隔壁巷子的刘氏，生了一个女儿。死那年金兵来了，没能跑出去。但那本手抄的《清明上河图注记》里夹着一片银杏叶，几百年后被人翻出来，夹在了某个博物馆的展柜里。

留下痕迹：一首酒后写在相国寺墙上的诗，被僧人用石灰盖住了，但石灰底下还能隐约看到字迹。
物品去向：打火机油尽被女儿埋在后院树下。指南针在逃难时遗失了。
```

## 游戏循环中的事件触发流程

```
  每轮 AI 对话结束
        ↓
  系统推进时间（#NEXT）
        ↓
  事件引擎：查 event 集合
        ↓
  命中事件？
   ├─ 是 → 注入 #EVENT（前因+场景）
   │         ↓
   │      AI 写叙事 = 场景展开（含该事件的世界面貌）
   │         ↓
   │      玩家回应（选项/自由输入）
   │
   └─ 否 → 无事发生，AI 写日常叙事
             ↓
         玩家回应
```

## 数据库写入映射

| 集合 | 写入时机 | 数据来源 |
|------|----------|----------|
| `system_message` | 游戏启动时 | 固定 System Prompt（一份） |
| `user_message` | 每次玩家操作 | #CHOICE / #FREE_INPUT 的内容 |
| `ai_message` | 每次 AI 回复 | 叙事全文 + 选项 + 状态标记 |
| `lives` | #SUMMARY 后 | 系统解析 SUMMARY 提取痕迹 + 物品去向 |
| `meta_message` | 跨世时 | 上一世关键物品去向、未了之事 |

---

## 跨世痕迹数据流

```
一世结束时：
  AI 生成 #SUMMARY
  系统解析 SUMMARY，提取：
    - 痕迹：文字类/血脉类/物品类/感觉类
    - 关键物品去向
  写入 lives 集合

下一世开始时：
  系统从 lives 随机选 1-2 条（优先选同一朝代或邻近地区）
  提取痕迹写成「余响」文字，注入 #INIT
  AI 在开篇叙事中自然融入余响
```

## 限制与注意事项

- 只允许一个 system message（MiniMax 硬性限制）
- AI 不需要知道具体月份，用季节/节气暗示
- `max_tokens` ≥ 5000
- `<think>` tag 前端必须剥离
- 输出纯文本，不含 JSON
- 状态标记 [内容] 会被系统解析，格式要一致
- 跨世痕迹一世最多 1-2 次，自然流露
- 不要直接告诉玩家"这是上一世的你"
- 物品数量不定（每局从30件中随机抽10件选3件），系统会在 #INIT 中列出
- **事件不预设后果**——只提供情境，AI 只展开场景和感官
- **概率只决定"正在发生什么"**——低概率 = 罕见场景，不一定好也不一定坏
