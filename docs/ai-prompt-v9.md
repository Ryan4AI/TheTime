# 穿越日记 · AI 叙事 Prompt v9（最终版）

> 2026-06-01 04:15 定稿

## System Prompt（固定，只一条）

```
你是《穿越日记》的世界叙述者。玩家是一个从21世纪穿越到真实中国历史的人，身上可能带着一些这个时代不该有的东西。
玩家会经历完整的一生，然后死去，再次穿越到另一个时代——一次又一次。过往的每一世都在历史中留下细微痕迹。
每段叙事后用「」给1~3个选项，另起一行用[]标记有变化的状态。
叙事40~120字，用「你」称呼，只写感官，大白话，不要古文。
跨世痕迹一世最多一两次，不要直说。
死亡和总结不需要选项和状态标记。
```

## 调用方式

每轮收集：system prompt + 全部对话历史 → 发给 AI → AI 返回纯文本 → 写回 ai_message

## 对话历史中的消息内容

| 角色 | 内容 |
|------|------|
| user（初始） | 你的身份、带的物品、跨世余响 |
| user（日常） | 玩家选的选项文字 或 自由输入的文字 |
| user（推进） | 时间过去了（直接写，无标记） |
| user（事件） | 某地在发生什么事（直接写，无标记） |
| user（死亡） | 死因 |
| assistant | AI 返回的原始纯文本 |

## ai_message 结构

```
{
  "_id": "auto",
  "role": "assistant",
  "content": "叙事文字。「选项A」「选项B」「选项C」\n[年龄=新值 物品=新状态]",
  "createdAt": Date
}
```

## user_message 结构

```
{
  "_id": "auto",
  "role": "user",
  "content": "玩家选项 / 自由输入 / 时间推进 / 事件描述 / 死因 / 初始信息",
  "createdAt": Date
}
```

## 前端解析

根据 ai_message.content 判断各段：
- 行首为「 → 选项
- 行首为 [ → 状态标记
- 其他 → 叙事文字
