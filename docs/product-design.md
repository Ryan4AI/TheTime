## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名 | **穿越日记**（英文：TheTime） |
| 一句话 | *AI根据真实数据模拟生成剧情，带你穿越到真实的历史场景。* |
| 形态 | 微信小游戏（Canvas 2D） |
| 核心爽点 | **留名青史，进入历史名人榜，摆脱死神追杀** |
| 核心标签 | AI剧情 · 穿越 · 历史真实数据 · 名人榜 · 死神追杀 |

### 核心循环

```
选物品 → 随机穿越身份 → AI叙事+选择 → 积累属性 → 冲击榜单
   ↓                                                       ↓
  下一世 ← 遗言+墓志铭 ← 死神追杀/寿终正寝 ← 超越历史名人
```

## 二、核心玩法循环

```
1. 选最多3件现代物品和性别
2. 按历史人口统计概率随机生成穿越年份/城市/年龄/阶层/职业
3. AI根据历史真实事件生成剧情场景，针对每个场景给玩家不同的选项
4. 玩家通过点击选项或输入文字做出决策
5. AI根据玩家输入决策生成下一阶段剧情，在生成剧情时会同时生成平行的多种剧情并给出在真实历史场景下每种剧情可能发生的概率，每种剧情包含各自要给玩家抉择的新选项，由系统通过随机概率选择呈现给玩家的剧情
6. 玩家针对新剧情进行新的决策，进入4-5的循环直至这一世死亡
7. 玩家留遗言，AI总结这一世历程并生成墓志铭
8. 回到1，进入下一世轮回
```

### 关键原则

**AI的权责：**
- ❌ 不能决定事件结果（系统投骰子决定）
- ❌ 不能编造历史事件（必须基于真实数据库）
- ✅ 负责撰写叙事文本、生成选项、输出属性变化和状态标记

**系统权责：**
- 按历史人口权重投骰子，决定玩家穿越身份
- 查询事件数据库，并将当前的真实历史事件作为上下文输入给AI
- 对AI生成的多个平行剧情投骰子，决定玩家被随机到哪一个
- 管理系统状态：9项属性、年龄、健康、金钱、物品、地点、身份、阶层
- 计算榜单排名、死神危险度
- 判定玩家是否死亡

## 三、属性体系

### 3.1 通用属性（4个）

| 属性 | 说明 | 范围 | 作用 |
|------|------|------|------|
| 声望 | 名气、人望 | 0-10000 | 所有榜单通用权重 |
| 财富 | 金钱、资产 | 0-10000 | 富商榜排序依据 |
| 学识 | 知识、学问 | 0-10000 | 文豪榜/全能榜权重 |
| 颜值 | 外貌、魅力 | 0-10000 | 颜值榜排序依据 |

### 3.2 专属属性（5个）

| 属性 | 说明 | 范围 | 对应榜单 |
|------|------|------|----------|
| 医术 | 医疗技能 | 0-10000 | 名医榜 |
| 战功 | 军事成就 | 0-10000 | 名将榜 |
| 文采 | 文学才华 | 0-10000 | 文豪榜 |
| 政绩 | 治国能力 | 0-10000 | 能臣榜 |
| 义行 | 侠义行为 | 0-10000 | 义士榜 |

### 3.3 属性增长规则

详见 `docs/attribute-rules.md`，核心要点：
- 8大类行为（医疗/军事/文学/政治/侠义/商业/学习/社交）各有对应属性增长
- 乱世加成：战功/义行 ×1.5，财富 ×0.7
- 盛世加成：财富 ×1.3，战功 ×0.8
- 历史事件加成：参与重大战役战功×2，平定大乱战功×2.5 等

### 3.4 属性初始化

| 属性 | 初始化规则 |
|------|-----------|
| 声望 | 贵族 500 / 平民 100 / 贱籍 50 |
| 财富 | 出身阶层 × 城市经济系数 |
| 学识 | 城市识字率 × 100 |
| 颜值 | 随机 3000-7000 |
| 医术/战功/文采/政绩/义行 | 全部 0（从零积累） |

## 四、历史名人榜系统

### 4.1 十大榜单

| 榜单 | 类型 | 排序公式 | 人物数 |

| 富商榜 | 专业 | 财富×1.0 | 100 |
| 文豪榜 | 专业 | 文采×0.7 + 学识×0.3 | 100 |
| 能臣榜 | 专业 | 政绩×0.7 + 声望×0.3 | 100 |
| 义士榜 | 专业 | 义行×0.7 + 声望×0.3 | 100 |
| 全能榜 | 专业 | 声望+财富+学识+颜值 | 100 |
| 长寿榜 | 趣味 | 寿命 | 30 |
| 旅行家榜 | 趣味 | 游历城市数 | 30 |
| 颜值榜 | 趣味 | 颜值×1.0 | 30 |

### 4.2 历史人物数据

- 总人物数：790人（7专业×100 + 3趣味×30）
- 数据文件：`data/leaderboards.json`
- 每个人物含：排名、姓名、朝代、各属性值、综合分、完整介绍（称号/传记/成就/名言/上榜理由）

### 4.3 玩家上榜规则

1. **进入门槛**：综合分 > 榜单最低人物的综合分
2. **排名计算**：按公式计算综合分，插入榜单对应位置
3. **多榜可上**：玩家可同时出现在多个榜单
4. **动态排名**：每次属性变化后重新计算排名
5. **超越提示**：当玩家综合分超过某历史人物时，叙事提示"你的医术已超越华佗！"

### 4.4 榜单交互设计

**榜单入口页：**
- 10个榜单 Tab 切换
- 每个榜单显示 Top 100
- 玩家当前位置高亮（如"你排第23位"）
- 点击历史人物 → 详情页（传记/成就/名言）

**回合反馈：**
- 每回合结束显示属性变化
- 显示榜单排名变化（如"名医榜：第45位→第38位 ↑7"）
- 超越历史人物时特殊提示

## 五、死神追杀机制

### 5.1 核心设定

> 死神（命运之轮）暗中追杀玩家。玩家通过积累属性、留名青史来获得"历史庇护"，摆脱死神。

### 5.2 危险度系统

| 危险等级 | 触发条件 | 死神行为 |
|---------|---------|---------|
| 安全 | 综合属性 > 5000 或 上榜 | 不出现 |
| 潜伏 | 综合属性 3000-5000 | 偶尔制造小意外 |
| 逼近 | 综合属性 1000-3000 | 频繁制造危机事件 |
| 紧迫 | 综合属性 < 1000 | 每回合都有生命危险 |
| 锁定 | 连续3回合无属性增长 | 必定触发致命事件 |

### 5.3 历史庇护

- 每上榜一次 → 获得 1 层历史庇护（最多 5 层）
- 每层庇护降低一个危险等级
- 5 层庇护 = 永久安全（成为历史传奇）
- 历史庇护层数带入下一世（保留部分）

### 5.4 死神事件类型

| 类型 | 说明 | 应对 |
|------|------|------|
| 意外 | 坠马、溺水、中毒 | 属性检定（体质/反应） |
| 疾病 | 瘟疫、疑难杂症 | 医术检定 |
| 战乱 | 兵灾、匪患 | 战功/武力检定 |
| 阴谋 | 权贵陷害、背叛 | 声望/人脉检定 |
| 天灾 | 洪水、地震 | 运气检定 |

### 5.5 死亡与轮回

- 死亡后总结本世成就：获得属性、上榜情况、历史庇护层数
- 跨世积累，最终目标是成为传奇

## 六、游戏流程详解

### 6.1 入口页 → 选物品

**入口页（entry.js）**
- 暗色古风 UI，逐字动画"穿越日记"
- 穿越特效（时空隧道 + 轮盘合体）
- 点击 → 进入选物品或碑林

**选物品（selection.js）**
- 后台30件物品池，每局随机抽10件展示
- 玩家选最多3件带穿越
- 可自由填写物品（暂未实现）
- 选完 → 进入穿越特效（intro.js）

**物品设计原则：**
- 所有物品来自现代，具备跨时代实用价值
- 如：打火机、急救包、钢制匕首、打火石、望远镜、防水火柴、指南针、钓鱼套件、复合维生素、太阳能充电宝
- ❌ 不显示耐久数值，只通过AI叙事暗示状态（"火石快磨没了"）

### 6.2 穿越 → 身份生成

**穿越特效（intro.js）**
- 时空隧道动画 + 朝代轮盘旋转
- 同时调用云函数生成身份
- 完成后 → 展示身份卡

**身份生成算法：**
1. 遍历 `era_cities` 所有记录，按城市人口加权随机。名人彩蛋 → 极低概率穿越成名人（名人数量/城市人口）。
2. 选到时代 + 城市后，查 `era_meta` 拿朝代/皇帝/识字率/姓氏
3. 从 `era_age_dist` 加权随机年龄
4. 从 `social_structure` 加权随机社会阶层
5. AI根据时代/城市/性别/阶层生成名字

**身份卡展示字段：**
- 纪年（eraDisplay，如"北宋·崇宁元年"）
- 年龄、性别
- 身份（职业）
- 阶层
- 居所（城市）

### 6.3 AI 对话（game.js）

身份卡展示完成后 → **直接进入 AI 对话**

```
[system_message] ← 固定规则 + 动态状态（每轮更新）
    ↓
[ai_message] ← AI生成，含多个平行剧情分支（items数组）
    ↓
[system_message] ← 随机选中剧情
    ↓
[system_message] ← 如果月份变化，加入当月发生的历史事件
    ↓
[user_message] ← 玩家点击选项、自由输入或使用物品
```

**每轮 AI 调用流程：**
1. 拼装prompt和上下文，包含系统设定、前世概览、当世所有交互对话
2. 调用chat API生成下一条AI消息
3. 剥离 `<think>` 推理 tag
4. 解析剧情 + 选项 + 属性变化（patch）+ 状态标记
5. 渲染剧情文本 + 属性变化动画
6. 显示选项按钮及自由输入框
7. 玩家选择 → 下一轮

### 6.4 消息系统（三个集合）

- **system_message**: 系统规则、随机选中结果、状态变化、当月历史事件
- **ai_message**: AI输出的多个平行剧情分支，含系统状态变化和给玩家的选项
- **user_message**: 玩家操作输入（点击选项、自由输入、使用物品）

### 6.5 历史事件

历史事件分成影响全国和影响城市2种。对于影响全国的历史事件，一般需要5-10轮对话完成；对于影响城市的历史事件，一般需要3-5轮对话完成；如果当月无历史事件，则1-2轮对话快速推进。

### 6.6 蝴蝶效应

玩家在某一历史事件的行为，可能会对后续的历史事件产生影响。当融入玩家的行为干预时，历史事件可以适当创作改编，不需要完全与真实情况贴合。

### 6.7 死亡与跨世

**死亡判定：**
- 自然死亡：玩家健康随年龄衰减
- 意外死亡：战争、瘟疫、饥荒、凶杀等（健康值骤降为0）
- 寿终：寿命系统（lifespan）归零

**跨世痕迹系统（四层设计）：**

第一层：文字留痕
- 上一世写过诗 → 这一世在茶馆听人吟唱
- 上一世留下日记/信件 → 在旧书摊偶然翻到
- 史书记了一笔"某年某月，有异人献奇物于朝"

第二层：血脉回响
- 上一世收过徒弟/养过孩子 → 碰到自称某派传人的人
- 上一世修过桥 → 那桥还在，当地人称"古时候一位奇人修的"

第三层：物品轮回
- 打火机油尽被丢弃 → 古董摊上的"奇怪铁匣"
- 指南针传世 → 道观里的风水罗盘
- 急救包 → "仙人所遗药囊"

第四层：念念不忘
- "走进这座城，莫名觉得某条巷子很熟悉"
- "那个卖饼的老汉说话腔调让你心里一动"

### 6.8 多玩家互动

基础机制：玩家可以看到所有玩家的墓碑。
后续可考虑：穿越成就、排行榜、玩家在平行世界中互动。

## 八、边缘场景与约束

### 时间
- ❌ 不写具体月份数字
- ✅ 用季节/节气暗示时间流逝

### 物品
- ❌ 物品不消失（除非叙事明确写了）
- ❌ 不显示数值状态
- ✅ 物品状态在 AI 叙事中用语言暗示

### AI 调用
- ❌ 推理链输出不可关闭（MiniMax-M2.7）
- ✅ 前端用正则剥离 `<think>` tag
- ✅ `max_tokens` ≥ 5000
- ❌ MiniMax 只允许一个 system message（2013 错误）
- ✅ 云函数通过跳过 system 或映射为 user 避免冲突

### 画面规范
- ❌ 不要古文风（"吾"、"汝"、"之乎者也"）
- ✅ 白话讲故事风格
- ❌ 不要太多"休息"、"继续前行"、"你又上路了"

### 10.3 API 配置

- **叙事引擎**: MiniMax-M2.7（云函数）
- **定位引擎**: DeepSeek v4 Flash（用于命名等轻量任务）
- **MiniMax 约束**: 只允许一个 system message
- **MiniMax 400错误**: 通常是限流/内容过滤/临时故障，非prompt长度问题
- 前端不直接调API，全部通过微信云函数中转
