## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名 | **穿越日记**（英文：TheTime，先生定名） |
| 一句话 | *带三件现代物品穿越到真实中国历史——AI记叙，概率定命，一生一诺* |
| 形态 | 微信小游戏（Canvas 2D，非小程序） |
| 目标 | 让玩家在真实历史中体验"如果我穿越回去会是什么样" |
| 核心标签 | AI叙事 · 概率驱动 · 历史真实数据 · 跨世轮回 |

## 二、核心玩法循环

```
带3件物品 ←→ 穿越到真实朝代
        ↓
  按历史人口概率随机分配时代/城市/年龄/阶层
        ↓
  AI写入世叙事（睁眼瞬间）
        ↓
  ┌→ AI叙事推进（含事件/日常）
  │   ↓
  │  玩家：选预设选项 或 自由输入
  │   ↓
  │  系统查历史概率/事件 → 决定结果
  │   ↓
  │  AI写结果叙事（含状态标记）
  │   ↓
  └── 循环直到死亡 ──→ 跨世轮回（有上一世痕迹）
                           ↓
                      回到选物品
```

### 关键原则

**AI的权责：**
- ❌ 不能决定事件结果（系统投骰子决定）
- ❌ 不能编造历史事件（必须基于数据库）
- ❌ 不能写心理活动
- ❌ 不能写"最后一刻"的煽情文字
- ✅ 只能写叙事——感官所见、季节变化、NPC反应
- ✅ 通过 `[状态标记]` 汇报物品状态变化

**系统权责：**
- 按历史人口权重投骰子，决定身份/时代/城市
- 查询事件数据库，触发真实历史事件
- 对玩家选择投骰子，决定后果
- 管理状态：年龄、金钱、体质、地点、身份、标记
- 判定死亡（自然/事件致死）

## 三、游戏流程详解

### 3.1 入口页 → 选物品

**入口页 （entry.js）**
- 暗色古风 UI，逐字动画"穿越日记"
- 穿越特效（时空隧道 + 轮盘合体）
- 点击 → 进入选物品或轮回记录

**选物品 （selection.js）**
- 后台30件物品池，每局随机抽10件展示
- 玩家选3件带穿越
- 可自由填写物品（待实现）
- 选完 → 进入穿越特效（intro.js）
- **已实现：** ✅ 30抽10选3 卡片布局

**物品设计原则：**
- 所有物品来自现代，具备跨时代实用价值
- 如：打火机、急救包、钢制匕首、打火石、望远镜、防水火柴、指南针、钓鱼套件、复合维生素、太阳能充电宝
- ❌ 不显示耐久数值，只通过AI叙事暗示状态（"火石快磨没了"）
- ❌ 不设独立物品面板，物品操作用法嵌入叙事和自由输入中

### 3.2 穿越 → 身份生成

**穿越特效 （intro.js）**
- 时空隧道动画 + 朝代轮盘旋转
- 同时调用云函数 `generate_identity` 生成身份数据
- 完成后 → 展示身份卡

**身份生成算法：**
1. 遍历 `era_cities` 所有记录，按城市人口加权随机
2. 选到时代 + 城市后，查 `era_meta` 拿朝代/皇帝/识字率/姓氏
3. 从 `era_age_dist` 加权随机年龄
4. 从 `social_structure` 加权随机社会阶层 + 随机职业
5. 若城市有名人彩蛋 → 极低概率穿越成名人（名人数量/城市人口）
6. AI（DeepSeek v4 Flash）根据时代/城市/性别生成名字
7. 名字生成失败 → 本地姓氏池+常用名组合降级

**身份卡需展示的字段：**
- 纪年（eraDisplay，如"北宋 · 崇宁元年"）
- 年龄、性别、婚姻状态
- 身份（职业）
- 阶层
- 居所（城市）
- 城市人口
- 城市描述
- 名人彩蛋（穿越成了历史人物）

### 3.3 AI 叙事对话（game.js）

身份卡展示完成后 → **直接进入 AI 对话**（无独立的 generate_story 云函数）

**每轮 AI 调用流程：**
1. 拼装 messages 数组
2. 调用 MiniMax 或 DeepSeek API
3. 剥离 `<think>` 推理 tag
4. 解析叙事 + 选项 + 状态标记
5. 渲染叙事文本
6. 显示选项按钮（或自由输入框）
7. 玩家选择 → 下一轮

**输出格式（已定稿）：**
- **叙事文本**：纯文字，60-120字描述当前场景
- **选项标记**：`「选项A説明」「选项B説明」「选项C説明」`
  - 选项放入「」中，AI 也使用这个标记在叙事中插入预设选项
  - 玩家从选项中选择（或自由输入其他内容）
- **状态标记**：每段叙事末尾另起一行，`[年龄 物品=状态]` 格式
  - 只写有变化的项
  - 例：`[28岁 打火机=油快尽了 铜钱=300文]`
- **系统标记**（AI调用时的控制指令）：
  - `#INIT`：玩家穿越入世，AI写睁眼的第一个画面
  - `#NEXT`：往前走一段时间（不要写具体月份）
  - `#CHOICE`：玩家做了选择
  - `#FREE_INPUT`：玩家自由输入了内容
  - `#DEATH`：玩家死亡，AI写死亡叙事
  - `#SUMMARY`：生命结束，AI写人生总结

### 3.4 消息系统（三个集合）

每轮对话产生两条记录：

| 集合 | 内容 | 写入时机 |
|------|------|----------|
| `system_message` | 系统 prompt（固定世界观） | 游戏开始时写入一次 |
| `user_message` | 玩家输入（选项/自由输入） | 玩家操作后写入 |
| `ai_message` | AI 叙事输出 | AI 回复后写入 |

AI 调用的上下文 = 最近 N 轮的 user + ai message，加上 system prompt。

### 3.5 死亡与跨世

**死亡判定：**
- 自然死亡（年龄增长到时代预期寿命）
- 事件致死（战争、瘟疫、饥荒、凶杀等）
- 由系统判定，AI 只负责写死亡叙事

**跨世轮回：**
- 死亡 → 回到选物品界面
- 每世结束时生成 **「人生总结」**
- 跨世时可以看见上一世的影子
  - 上一世写的诗出现在县志/古籍中
  - 上一世认识的人的后代与你相遇
  - 上一世建的建筑成为古迹
  - 上一世的事迹被记录在历史上
  - （待细设计）
- 跨世数据存放在 `lives` + `meta_message` 集合

## 四、数据库结构（实际部署）

### 4.1 数据集合（有 schema 验证）

| 集合 | 数据量 | 用途 |
|------|--------|------|
| `era_meta` | 按年份 | 时代元数据（朝代、皇帝、识字率、姓氏池） |
| `era_cities` | 按年份+城市 | 城市人口、描述、名人 |
| `era_age_dist` | 按年份+年龄 | 年龄分布权重（0-80岁每岁一行） |
| `social_structure` | 按年份+阶层 | 社会阶层权重 + 职业池 |
| `event` | 按年份+月份+城市 | 历史事件（标题、描述、类型、影响范围） |

## 六、Design Decisions（已定案）

| # | 决策 | 具体内容 | 定案时间 |
|---|------|----------|----------|
| 1 | 物品不显示数值 | 不显示耐久/用量，只通过叙事暗示 | 05-30 |
| 2 | 不搞兜底叙事 | AI失败就重试，不准备 fallback | 05-30 |
| 3 | 自由输入由AI判定 | 玩家自由输入的内容由AI判断结果 | 05-30 |
| 4 | 统计数据不用AI生成 | 人口数据等直接展示在卡面上 | 05-30 |
| 5 | 项目定位"穿越日记"而非"出生叙事" |  零岁出生 → 带物品在已有年纪穿越醒来 | 05-30 |
| 6 | 概率驱动而非AI驱动结果 | 系统投骰子定命，AI只负责写叙事 | 05-30 |
| 7 | 小游戏非小程序 | AppID 注册为小游戏 | 05-31 |
| 8 | 名字由AI生成 | male/femaleNames 改为可选 | 05-31 |
| 9 | 身份卡 → 直接AI对话 | 不需要独立的 generate_story | 06-01 |
| 10 | 无 atmosphere 字段 | 已从设计中去除 | 06-01 |
| 11 | 输出纯文本 + 标记 | 非 JSON 格式叙事 | 06-01 |
| 12 | 跨世需要具体巧思 | 上一世痕迹（诗/人/建筑） | 06-01 |
| 13 | 按角色分三个 message 集合 | system/user/ai 分三表 | 06-01 |

## 八、边缘场景与约束

### 时间
- ❌ 不写具体月份（不用"一个月"、"这年"、"过了XX"）
- ✅ 用季节/节气暗示时间流逝

### 物品
- ❌ 物品不消失（除非叙事明确写"打火机掉河里了"）
- ❌ 不显示数值状态
- ✅ 物品状态在 AI 叙事中用语言暗示

### AI调用
- ❌ 推理链输出不可关闭（MiniMax-M2.7）
- ✅ 前端用正则剥离 `<think>`
- ✅ `max_tokens` ≥ 5000
- ❌ 只允许一个 system message（MiniMax）

### 画面规范
- ❌ 不要古文风（"吾"、"汝"、"之乎者也"）
- ✅ 白话讲故事风格
- ❌ 不要太多"休息"、"继续前行"、"你又上路了"
